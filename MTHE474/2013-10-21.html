Huffman Optimal Coding Algorithm for DMSÂ¶

Corollary

Huffman - mid-1950s as a Masters Student 3-4 page paper

Consider a DMS \(\mathcal X=\{a_1,...,a_M\}\) with source symbol probabilities \(p_1\leq p_2\leq ... p_M\)

Consider a reduced source \(\mathcal Y\) obtained from \(\mathcal X\) by combining the two least likely symbols \(a_{M-1},a_{M}\) into an equivalent symbol \(a_{M-1,M}\) (with probability \(p_{M-1}+p_M\))

\[\mathcal Y = \{a_1,...,a_{M-1,M}\}\]

with probabilities \(p_1\leq p_2\leq ... (p_{M-1}+p_M)\)

Suppose we have an optimal (binary prefix) code \(\mathcal C_2\) for the reduces source \(\mathcal Y\). We now construct a code \(\mathcal C_1\) for the original source \(\mathcal X\) as follows:

  • The codewords for symbols \(a_1,a_2,...,a_{M-2}\) are exactly the same as those in \(\mathcal C_2\)

    \[\begin{split}f_1(a_i)=f_2(a_i) & i=1,...,M-2\end{split}\]

  • The codewords for symbols \(a_{M-1}, a_M\) are formed by appending a 0 and a 1, respectively to the codeword for symbol \(a_{M-1,M}\) in \(\mathcal C_2\):

    \[\begin{split}f_1(a_{M-1})=f_2(a_{M-1,M})0\\ f_1(a_M) = f_2(a_{M-1,M})1\end{split}\]

\(\mathcal C_1\) is an optimal code for the original source \(\mathcal X\).

Note

There is a recursive algorithm underlying here for the construction of optimal binary prefix codes:

“Repeatedly apply the Huffman result to the source until you are left with a reduced source of size 2 - that is, \(M-2\) times.

An optimal binary code for this reduced source must consist of two codewords: ‘0’, and ‘1’. Then proceed backwards constructing optimal codes for each reduced source until you reach the original source.”

Examples

  1. \[\begin{split}\mathcal X = \{a_1,a_2,a_3,a_4\}, & p_1=\frac{1}{2},p_2=\frac{1}{4},p_3=p_4=\frac{1}{8}\end{split}\]

    we have \(p_1\geq p_2 \geq p_3 \geq p_4\)

    Thus, the Huffman Code maps:

    \[\begin{split}f:\mathcal X & \to \{0,1\}^*\\ a_1 &\to 0\\ a_2 &\to 10\\ a_3 &\to 110\\ a_4 &\to 111\end{split}\]

    This Huffman code is absolutely optimal, since it achieves the entropy

  2. \[\begin{split}\mathcal X = \{a_1,...,a_6\}, & p_i=(\frac{3}{8},\frac{1}{4},\frac{1}{8},\frac{1}{8},\frac{1}{16},\frac{1}{16})\end{split}\]
    \[\begin{split}f:\mathcal X & \to \{0,1\}^*\\ a_1 &\to 1\\ a_2 &\to 011\\ a_3 &\to 010\\ a_4 &\to 001\\ a_5 &\to 0000\\ a_6 &\to 0001\end{split}\]
    \[\begin{split}\bar R=\frac{\bar l}{1}=\frac{19}{8}=2.375 \frac{\text{code bit}}{\text{source symbol}}\\ H(X)=2.28\text{ bits}\end{split}\]

    Note

    \[H(X)\leq \bar l\leq H(X)+1\]

  3. nth-order Huffman codes:

    By encoding larger blocks of source symbols, we can improve Huffman code efficiency.

    \[f:\mathcal X^n\to \{0,1\}^*\]

    \(n=2\) Now source \(\mathcal X^2=\{a_1a_1,a_1a_2,a_1,a_3,...,a_4a_4\}\)

    Exercise

    Construct 2nd order Huffman Code

    \[\begin{split}f: \mathcal X^2\to \{0,1\}^*\\ \bar R=\frac{\bar l}{n}=\frac{\bar l}{2}=\frac{3.859}{2}=1.9297\frac{\text{code bit}}{\text{source symbol}}\end{split}\]
    \[\begin{split}H(X)\leq\bar R< H(X)+\frac{1}{2}\end{split}\]

Observations

  • Huffman Codes are not unique - i.e there is more than one possible Huffman code (ex. by inverting all the code bits of a Huffman code, you get another Huffman code, or by resolving probability ties in different ways in the Huffman algorithm, you get different Huffman codes.) But, the all have identical, minimal, \(\bar l\) (and \(\bar R\)).

  • One can also obtain optimal codes which are not Huffman (e.g. by interchanging two codewords of the same length in a Huff. code, you get another non-Huffman, but optimal code.)

  • We can also get an optimal suffix code from a Huffman code (which is prefix) by reversing codewords:

    Huffman Suffix
    \(a_1\to 0\) \(a_1\to 0\)
    \(a_2\to 10\) \(a_2\to 01\)
    \(a_3\to 110\) \(a_3\to 011\)
    \(a_4\to 111\) \(a_4\to 111\)

  • For an nth order Huffman code:

    \[\begin{split}f:\mathcal X^n\to\{0,1\}^*\\ H(\mathcal X)\leq \frac{H(X^n)}{n}\leq \bar R=\frac{\bar l}{n}<H(X^n)+\frac{1}{n}\end{split}\]

    for a stationary source.

    As \(n\) increases, \(\bar R\to H(\mathcal X)\)

    but, complexity grows exponentially with \(n\) (\(\sim |\mathcal X|^n\)) and encoding delay (\(n\)).
