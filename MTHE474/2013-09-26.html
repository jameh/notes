Lossless Fixed Length (Block) Source Coding (Data Compression)¶

for Discrete Memoryless Source (DMS).

Definition

Given integers \(n\) and \(k\) (\(k=k(n)\)), then a \((k,n)\) D-ary block encoder for a source \(\{X_i\}^\infty_{i=1}\) with finite alphabet \(\mathcal X\) is a map:

\[f: \mathcal X^n \to \{0,1,...D-1\}^k\]

Where \(D\geq 2\), integer.

The encoder represents \(n\) consecutive symbols from the source by \(D\) - ary \(k\) -tuples, which we call codewords. (each codeword contains \(k\) \(D\) -ary code symbols.)

wlog, we will deal with the case \(D=2\):

\[f: \mathcal X^n \to \{0,1\}^k\]

In this case, a code symbol is a code bit.

(Code compression rate)

\[R = \frac{k}{n}\]

where the unit is \(\frac{[code symbols (code bits)]}{some symbol}\).

Problem¶

We want to construct a block encoder for a DMS:

\[f: \mathcal X^n \to \{0,1\}^k\]

coupled with a decoder:

\[g: \{0,1\}^k \to \mathcal X^n\]

such that

\[\begin{split}P_e:=P[g(f(x_1,...,x_n)) \neq (x_1,...,x_n)] < \epsilon\end{split}\]

for any \(\epsilon>0\), where \(P_e\) is the probability of decoding error.

and

\[R=\frac{k}{n}\]

is as small as possible.

Note

\(\mathcal X^n\) could be anything like the English language for example based on the English alphabet.

Note

Using block codes, not acutally Lossless compression; but asymptotically Lossless compression.

Question:

How small can \(R\) be for \(\epsilon\) arbitrarily small?

Note

Block code \((k,n)\) is the pair \((f,g)\). (encoder, decoder).

codebook:

\[e = f(\mathcal X^n) = \text{set of all the codewords}\]

Definition

A \((k,n)\) binary block source code is uniquely decodable (UD) - or lossless - if every source \(n\) -tuple is assigned to distinct codewords (binary \(k\) -tuple), and all the codewords are used.

i.e. \(f\) is bijective.

\(\iff\) \(f\) is invertible and \(g=f^{-1}\).

For a UD code, \(P_e=0\), its rate is given by:

\[|\mathcal X|^n = 2^k\]\[\iff R \frac{k}{n} = \log{2} |\mathcal X| \text{ code bits/source symbol}\]

Question:

Can we do better? i.e. have smaller compression rates with block source codes?

Answer:

Yes, if we give up Unique Decodability for finite values of \(n\). and require it to hold as \(n \to \infty\) and take into account the statistical properties of the source (think AEP).

Theorem

Shannon’s Lossless Fixed Length Source Coding Theorem for DMSs

Note

Check Appendix A in online notes for background on infimum/supremum stuff.

Consider a DMS \(\{X_i\}_{i=1}^\infty\) with alphabet \(\mathcal X\), generic pmf \(p_X\) and entropy:

\[H_D(X) = -\sum_{x \in \mathcal X} p_X(x) \log{D}p_X(x)\]

Note

\(H_D\) is in base \(D\).

  1. Forward (Achievability ) Part:

For any \(a < \epsilon < 1\) and \(\alpha > 0\), there exists a sequence of \(D\) -ary block codes \((k,n)\) for the source such that:

  1. \(\frac{k}{n}<H_D(X) + \alpha\)

and

  1. \(P_e<\epsilon\)

for \(n\) sufficiently large.

  1. (Strong) Converse Part

For any \(0<\epsilon<1\) and any sequence of \(D\) -ary block codes \((k,n)\) for the source with rate:

\[\begin{split}R=\frac{k}{n} < H_D(X)\end{split}\]

and \(n\) sufficiently large, we have that:

\[\begin{split}P_e > 1-\epsilon\end{split}\]

Conclusion:

No matter how small the prescribed \(\epsilon\), the code rate can be brought close to \(H_D(X)\), but cannot go below it if we are to construct asymptotically lossless \(D\) -ary block codes.

Therefore, \(H_D(X)\) is the smallest block source coding rate for which \(\exists\) \(D\) -ary block codes for a DMS with asymptotically (in \(n\)) vanishing probability of decoding error.
